# Project Scope

## Project Name:
AutoReq — Automated Requirement Gathering System

## Domain / Product Type:
Software Engineering / Requirements Engineering (SaaS tool for dev teams)

## Primary Users / Stakeholders:
- Business Analysts
- Product Managers
- Developers
- QA Engineers
- Clients who provide requirement inputs

## MVP Capability:
Automatically extract functional and non-functional requirements from natural-language sources (e.g., user stories, GitHub issues, support tickets, app reviews) and produce a structured requirements specification.

## Top 3 Success Metrics:
1. Achieve at least 80% precision/recall in classifying Functional vs Non-Functional requirements.
2. Reduce time spent on manual requirement gathering by 30%.
3. Positive feedback from ≥70% of stakeholders during pilot testing.

## Data Sources for MVP:
- Sample dataset (`data/sample_dataset.csv`)
- GitHub issues and Jira tickets
- Small set of app store reviews
- Simulated stakeholder interview transcripts
